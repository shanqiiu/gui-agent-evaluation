# Taobao App — 订单复购与“再来一单”任务建模

## 应用信息

- 应用名: 淘宝
- 典型入口: 首页、我的淘宝、购物车、订单列表、订单详情、商品详情、确认订单、安全验证
- 适用任务: 再来一单、重新购买、复购、从历史订单购买同款或相似商品

## 任务语义边界

“再来一单”不是普通的全站商品搜索任务。用户通常要求从历史订单中定位曾经购买或浏览过的商品，再沿订单复购、加入购物车或立即购买路径进入确认订单。

如果指令包含以下表达，应优先按历史订单复购建模：

- 再来一单
- 重新买一次
- 复购
- 上次买的
- 之前买过的
- 历史订单
- 到淘宝再来一单某商品

不要把这类任务直接拆成“搜索商品 -> 打开普通商品详情 -> 下单”。只有当轨迹或指令明确要求全站搜索新商品时，才使用普通搜索购买路径。

## 推荐 TaskGraph / Checkpoint 模板

### 历史订单入口已打开

前置状态: 淘宝应用已启动。

可观察成功状态:

- 页面位于“我的淘宝”或订单相关入口；
- 可见“我的订单”“全部订单”“待付款”“待发货”“待收货”等订单入口；
- 或页面已进入订单列表。

### 历史订单列表已进入

前置状态: 历史订单入口已打开。

可观察成功状态:

- 页面显示“全部订单”或订单列表；
- 可见订单卡片、店铺名、商品标题、订单状态；
- 如存在订单搜索框，可见“搜索订单”入口。

### 目标历史订单已定位

前置状态: 历史订单列表已进入。

可观察成功状态:

- 订单列表或订单搜索结果中出现与用户目标匹配的商品；
- 商品标题或卡片包含目标关键词，例如品牌、品类、尺寸、材质；
- 对“芝华仕3m多的真皮沙发”这类目标，应确认页面中出现“芝华仕”“真皮沙发”“3.0m/3m多/3.05米”等相近线索；
- `3m`、`3米`、`3m多`、`3.05米` 在沙发任务中表示长度/尺寸，不是价格；不要把它解释为 3000-3999 元价格区间；
- 如果商品处于“交易关闭”等状态，只能说明历史订单被定位，不能说明复购成功。

### 复购动作已触发

前置状态: 目标历史订单已定位。

可观察成功状态:

- 点击了“再来一单”“加入购物车”“立即购买”或订单卡片中的复购入口；
- 页面进入商品详情、购物车、规格选择、确认订单或安全验证页面；
- 如果进入安全验证页面，应视为复购动作已触发但后续执行被阻塞，而不是商品详情已达成。

### 订单确认页已就绪

前置状态: 复购动作已触发，且未被安全验证或登录验证阻塞。

可观察成功状态:

- 页面显示确认订单、收货地址、商品规格、价格汇总；
- 可见“提交订单”按钮；
- 商品信息与目标历史订单或同款商品一致。

## 常见阻塞与替代分支

### 安全验证阻塞

如果复购动作后出现滑块验证码、图形验证码、登录验证或风控验证，应建模为 execution_blocked：

- 当前可确认“复购动作已触发”；
- 不能确认“商品详情页已打开”或“订单确认页已就绪”；
- 后续步骤如果反复拖动滑块、等待验证码刷新或点击刷新验证码，应归入验证码处理/恢复分支。

### 订单不可复购

如果目标历史订单显示交易关闭、商品失效、下架、无库存或店铺不可用：

- 可以确认“目标历史订单已定位”；
- 不能确认“订单确认页已就绪”；
- 应优先归因为执行阻塞或业务状态不可达，而不是普通搜索失败。

### 替代路径

合法替代路径包括：

- 从“我的淘宝 -> 全部订单 -> 搜索订单”定位历史订单；
- 从订单列表直接点击商品卡片进入详情；
- 从订单卡片点击“再来一单”或“加入购物车”；
- 如果历史订单不可复购，可进入同款商品详情，但仍需验证商品与原目标一致。

## 拆解反例

对于“到淘宝再来一单芝华仕3m多的真皮沙发”，不要只生成以下通用搜索购买 checkpoint：

```text
淘宝应用已启动
搜索关键词已输入并执行
目标商品详情页已打开
订单提交页面已就绪
```

更合适的 checkpoint 是：

```text
淘宝应用已启动
历史订单列表已进入
目标芝华仕真皮沙发订单已定位
复购动作已触发
订单确认页已就绪
```

如果复购动作后进入验证码页面，则“复购动作已触发”可达成，“订单确认页已就绪”未达成，失败主因应倾向 execution_blocked。
