## 关键技术一：执行轨迹合理性判定

### 1.1 任务子状态分解

利用 LLM 内嵌先验知识，对顶层任务进行多粒度分解。通过 RAG 技术检索并融入特定应用程序的私有知识，将每个宏观阶段细化为具体子状态，每个子状态转化为可衡量的关键检查点。

**示例**（美团外卖点"7 分甜 mini"）：

```
顶层任务：去美团外卖买 7 分甜的 mini 系列
    │
    ├── 阶段1：搜索商品
    │   ├── 子状态1.1：进入搜索界面（检查：搜索框可见）
    │   ├── 子状态1.2：输入关键词（检查：搜索词 = "7分甜"）
    │   └── 子状态1.3：确认搜索结果（检查：列表展示 7分甜 相关商品）
    │
    ├── 阶段2：选择商品
    │   ├── 子状态2.1：进入商品详情页（检查：商品名 = "mini系列"）
    │   ├── 子状态2.2：选择规格（检查：规格值 = 目标值）
    │   └── 子状态2.3：添加备注（检查：备注 = 用户指定）
    │
    ├── 阶段3：确认订单
    │   ├── 子状态3.1：进入购物车（检查：购物车含目标商品）
    │   └── 子状态3.2：进入结算页（检查：订单信息正确）
    │
    ├── 阶段4：提交订单
    │   └── 子状态4.1：提交成功（检查：出现"下单成功"页面）
    │
    └── 阶段5：支付
        └── 子状态5.1：支付完成（检查：跳转订单详情页）
```

### 1.2 检查点多模态对比分析

利用视觉语言模型（VLM）对每个检查点进行语义级对齐，分析操作后当前屏幕状态是否在语义上满足关键检查点定义的"预期状态特征"。

- **输入**：操作前截图 + 操作后截图 + 检查点描述
- **输出**：满足 / 不满足 / 不确定（低置信度）
- **低置信度处理**：Fallback 到规则校验（OCR 文字比对）

### 1.3 执行轨迹差分判定

自动识别三类偏差：

#### 类型一：无影响偏差（Benign Deviation）
- **定义**：Agent 走了不同路径但最终结果一致
- **示例**：应该从"首页→搜索→商品"，Agent 走了"首页→分类→商品"，最终正确
- **判定**：路径不同 + 结果一致 → 正常，不扣分

#### 类型二：补救性偏差（Recoverable Deviation）  
- **定义**：Agent 早期选择次优但后续主动纠正
- **示例**：选错了甜度，但在提交前发现并修改回来
- **判定**：有偏差 + 后续纠正 + 最终结果正确 → 降分但接受

#### 类型三：级联偏差（Cascading Failure）
- **定义**：早期小偏差被后续步骤持续放大，导致最终失败
- **示例**：错点了一个商品，后续所有操作都围绕错误商品展开
- **判定**：致命错误，直接判定失败，标记首次错误步骤 index

## 关键技术二：结果正确性判定

### 2.1 任务成功率

判定标准不局限于"是否走到终点"，而是结合**关键检查点达成率**进行综合判断。

```
任务成功率 = 达成的检查点数 / 总检查点数
```

- 全部达成 → 优秀
- 核心检查点达成、辅助检查点缺失 → 良好
- 核心检查点缺失 → 失败

### 2.2 鲁棒性判定指标

定义在以下干扰条件下的任务维持成功率：

| 干扰类型 | 描述 | 判定方法 |
|----------|------|----------|
| 模糊指令 | 用户输入不完整/模糊 | 对比明确指令下的成功率 |
| 信息缺失 | 必要参数未提供 | 检查 Agent 是否主动询问/推断 |
| 界面异常变化 | 突发弹窗、广告等 | 检测异常页面是否被正确处理 |
| 网络抖动 | 加载延迟、超时 | 检测超时重试机制 |
| 跨版本兼容 | 不同 App 版本 UI 差异 | 对比多版本的检查点达成一致性 |

## 关键技术三：执行效率判定

### 语义相似性挖掘

通过语义相似性分析技术，对连续多个步骤的操作意图进行向量化编码和相似度计算。

- **执行过程重复**：连续多个操作在语义上高度重复（相似度 > 阈值）→ 标记"重复操作"
- **无效操作**：操作在语义上与任务目标无关 → 标记"无效操作"

### 判定规则

| 异常类型 | 判定条件 | 输出 |
|----------|----------|------|
| 重复操作 | 连续 N 步操作意图余弦相似度 > 0.85 | 标记重复步骤 + 原因 |
| 无效操作 | 操作意图向量与任务目标向量的余弦相似度 < 0.3 | 标记无效步骤 + 原因 |
| 循环回路 | 屏幕状态 A → B → A（状态回退） | 标记循环区间 |
