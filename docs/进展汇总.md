# GUI Agent 测试判定——工作进展概要

## 技术基座

参考达尔文实验室的项目能力：基于 Qwen-VL-7B 微调，提供 GUI Agent 执行轨迹的功能完备性判定与功能可用性判定两项核心能力，作为本项目的技术起点。

## 调研进展

追踪 17 篇 2024-2026 顶会论文（ICML 2026、AAAI 2026、CVPR 2026），核心收获：

- RoTS（ICML 2026 Spotlight）：11 类 Error Taxonomy，执行层 UI 误操作占比 42.9%，规划层恢复率 <20%
- HalluClear：8 类 GUI 专属幻觉分类，证实感知幻觉是级联失败的根因
- Completion-Correctness Gap：Agent 完成率 96% 但正确率远低于此，验证了项目"不依赖自报、客观判定检查点达成"的路线
- 项目提出的差分偏差三分类（无影响/补救性/级联偏差）已得到上述学术研究的验证。

## 当前状态

### 已完成

**判定服务（Darwin 集成）**
- FastAPI 服务（port 20025），`/check_e2e` 端到端序列判定
- 达尔文 VLM+LLM 判定：AB 页面跳转、意图判定、Plan 覆盖
- 重复动作检测器（`repeated_action_detector.py`）：规则化读取 Darwin AB 标签，检测等效动作/目标/无进展
- 规划失效检测器（`planning_failure_detector.py`）：四种子类型（`missing_required_step` / `premature_termination` / `fail_to_terminate` / `objective_or_plan_mismatch`）
- `ParsedAction` 模型新增 `content` 字段（兼容 type/set_text 动作）
- finished 步骤不计入实际动作数
- `step_level_instruction` 分隔符统一为 `->`

**数据处理管线**
- `reorg_screenshots.py`：utg.json + catchDataTurnId 目录 → 扁平截图（按 turn ID 命名）
- `convert_to_check_e2e.py`：utg.json → /check_e2e payload 转换器，完整映射链路：
  - 从 `node.raw_item.directives` 提取动作类型、元素文本、操作坐标（`params.points` → node.bounds 中心）
  - 从 `node.image` REST URL 解析本地截图文件（支持 home/、end/、catchDataTurnIdN/ 等各类目录）
  - 截图三层兜底：自己节点 → 前节点 → 空
  - 动作类型归一化：edit→type, preCheckDone→do-nothing, scroll custom→scroll
  - `step_level_instruction` 生成（去重、do-nothing 过滤、坐标模糊化）
  - **新增 `--decompose` 参数**：支持 LLM+RAG 自动生成检查点分解作为 step_plan
- `send_payload.py`：已保存 payload 重发工具，支持路径 → base64 转换
- 批量断点续跑：已完成的 payload 自动跳过
- 10 项核心逻辑单元测试

**模块 A — 任务分解引擎（`src/decomposer/`）**
- `Decomposer` 类：LLM → 结构化检查点 JSON（name/required/preconditions/expected_state）
- `knowledge_store.py`：ChromaDB 向量存储 + 文档摄入（`--ingest` / `--query` CLI）
- App 知识库：`settings.md`（鸿蒙设置 App 页面流与关键元素）
- 已集成到 `convert_to_check_e2e.py`：通过 `--decompose` 参数自动生成 step_plan，替代人工标注
- 详见 `src/decomposer/README.md`
