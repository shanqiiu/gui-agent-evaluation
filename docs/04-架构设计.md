## 模块设计

### 模块 A：任务分解引擎
- **输入**：自然语言指令
- **处理**：LLM 多粒度分解 + RAG 注入私有知识
- **输出**：结构化检查点列表（含约束类型、预期状态描述、权重）

### 模块 B：检查点验证器
- **输入**：操作前后截图对 + 检查点描述
- **处理**：VLM 语义对齐（Qwen2.5-VL 系列）
- **输出**：达成/未达成/不确定 + 置信度

### 模块 C：效率分析器
- **输入**：操作流（action 序列）
- **处理**：操作意图向量化 → 余弦相似度 → 阈值判定
- **输出**：重复操作区间、无效操作列表、循环回路标记

### 模块 D：轨迹差分判定器
- **输入**：完整轨迹 + 检查点达成情况
- **处理**：三分类判定逻辑（无影响/补救性/级联）
- **输出**：偏差类型 + 首错步骤 index + 严重度评分

### 模块 E：综合评估器
- **聚合** 模块 B/C/D 的结果
- **加权** 计算总分（任务完成度 50% + 效率 25% + 异常处理 15% + 决策 10%）
- **生成** 结构化评估报告

## 技术选型

| 组件 | 推荐方案 | 备选方案 |
|------|----------|----------|
| VLM 语义对齐 | Qwen2.5-VL-7B | Qwen2-VL-7B, GPT-4o API |
| LLM 任务分解 | Qwen3-8B / GPT-4o | DeepSeek-V3 |
| 操作意图编码 | text-embedding-3-large | BGE-M3, jina-embeddings-v3 |
| RAG 知识库 | ChromaDB / LanceDB | Milvus Lite |
| 向量相似度 | 余弦相似度 | 欧氏距离, Dot product |
| 推理框架 | vLLM / SGLang | Ollama, llama.cpp |

## 执行风险与应对

| 风险 | 导致后果 | 应对策略 |
|------|----------|----------|
| VLM 对动态列表/滚动页面误判 | 检查点判定错误 | 重要检查点加置信度阈值；低置信度 fallback OCR |
| 检查点数量爆炸（复杂任务） | 推理成本过高 | 分级检查：粗粒度 OCR/规则过滤 → 只精判异常点 |
| 语义相似度阈值难以标定 | 重复/无效操作判定不准 | 人工标注 calibration 数据集确定阈值 |
| 级联偏差只能事后发现 | 在线审计不能实时定位 | 在线只做异常告警；完整三分类离线输出 |
