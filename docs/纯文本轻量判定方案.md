# 纯文本轻量判定方案（备选）

> 版本：v0.1 · 2026-07-08  
> 定位：不依赖 VLM 调用的轻量级异常判定，作为当前 VLM 全链路方案的降级/加速备选。

## 1. 定位

当前方案在每对截图上调用 VLM 做 AB 页面跳转判定，是链路中最重的计算开销。本方案砍掉 AB 判定，仅依赖 `seq_info` 中的结构化文本（动作类型、坐标、元素文本、方向），用规则实现重复动作和规划失效检测。

## 2. 输入

| 字段 | 用途 |
|------|------|
| `parsed_action.action_type` | 判断动作类型等效 |
| `parsed_action.start_box` / `end_box` | 坐标距离计算（重复检测） |
| `parsed_action.text` | 元素描述文本相似度（重复检测） |
| `parsed_action.direction` | 滑动方向一致性 |
| `step_level_instruction` | Plan 步骤分解 → 覆盖计算 |
| `index` | 步骤顺序 |

## 3. 可保留的判定

### 3.1 重复动作判定

不需要 VLM。当前检测器的输入都可以从 payload 直接获取：

| 当前依赖 Darwin 的字段 | 纯文本替代 |
|---|---|
| `ab_pages_result.action_des` | `parsed_action.text`（我们已从 directives 提取） |
| `ab_pages_result.label` | 删除此维度的判断（不再有"是否符合预期"） |
| `ab_pages_result.pagea/pageb_description` | 删除页面描述相似度，仅保留坐标+文本相似度 |
| `llm/vlm_intention_step` | `step_level_instruction` 逐项匹配动作文本 |

改动：降低 `_no_progress_between` 对 AB 标签的依赖，增加坐标距离和文本相似度的权重。

### 3.2 规划失效判定

不需要 VLM。改为纯规则：

| 子类型 | 纯文本判定逻辑 |
|--------|---------------|
| 遗漏必要步骤 | `step_level_instruction` 拆解 → 逐项检查 `seq_info[*].text` 是否匹配 |
| 提前终止 | 最后一步是 `finished` + Plan 覆盖率 < 阈值 |
| 未能终止 | 检测到`finished`后还有额外操作步骤 |
| 目标不一致 | 不判定（需要 VLM 意图判定） |

Plan 覆盖计算：将 `step_level_instruction` 的每个步骤名与 `seq_info[*].text` 做字符串相似度匹配，匹配成功计为覆盖。

## 4. 不可保留的判定

| 判定 | 原因 |
|------|------|
| 页面跳转是否正确（AB 判定） | 必须看图。Agent 点了一个按钮但跳到了错误页面——纯文本无法判断 |
| 整体意图是否达成（intention.label） | 需要 VLM 看图判断最终页面是否符合预期 |
| 目标不一致（objective_or_plan_mismatch） | 依赖达尔文意图判定结果 |

## 5. 逻辑改动量

| 文件 | 改动 |
|------|------|
| `repeated_action_detector.py` | `_no_progress_between` 移除 `ab_label` 维度的检查；`_build_steps` 的 `target` 字段改用 `parsed_action.text` |
| `planning_failure_detector.py` | 新增 `_match_plan_to_actions()` 用文本匹配替代 `llm/vlm_intention_step`；移除 `_is_objective_or_plan_mismatch` |
| `oracle_service.py` | 跳过 `ab_pages_validate()` 调用（或整个 `run_sequence` 替换为纯文本版本） |

## 6. 适用场景

- **快速预筛**：大批量样本中先筛出明显异常的，VLM 只跑可疑样本
- **离线回溯**：已有历史轨迹但 VLM 不可用时的补救分析
- **性能优化**：长序列（>20 步）的 VLM 滑动窗口开销过大时的降级
