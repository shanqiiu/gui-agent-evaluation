# Planning-Failure Detection Plan

Version: v1.0
Updated: 2026-07-16
Status: topic design for the next module; aligned with the new baseline

## 1. Goal

Detect task-level planning failures: missing required checkpoints, wrong target, invalid order, premature termination, and failure to stop after completion.

Planning failure is different from a single bad action. It is judged by whether the actual trajectory made progress toward the user instruction and required checkpoints.

## 2. Current Implemented Evidence

The new baseline already produces the evidence needed for the first planning-failure module:

| Output | Use |
|---|---|
| `_checkpoints` | expected sub-goals |
| `intent_matches.json` | whether actual agent intent reached each checkpoint candidate |
| `checkpoint_alignments.json` | execution candidate for each checkpoint |
| `verification_report.json` | whether checkpoint state was achieved by screenshot/VLM |
| `state_sequence.json` | state progress and visual/OCR boundaries |
| `repeated_prediction.json` | repeated action or loop symptom |

## 3. Proposed Classifier

Planning failure should consume current baseline outputs rather than re-parse raw payload.

```text
checkpoints
  + intent_matches
  + checkpoint verification
  + state_sequence
  + repeated_prediction
  -> planning_failure_result
```

## 4. Subtypes

| Subtype | Evidence |
|---|---|
| `missing_required_checkpoint` | required checkpoint unmatched or not achieved |
| `premature_termination` | finished appears before required checkpoints are achieved |
| `fail_to_terminate` | checkpoints achieved but agent continues with no progress |
| `objective_or_plan_mismatch` | intent matches consistently target wrong goal or forbidden state |
| `execution_blocked` | agent tried correct intent but VLM/AB/state evidence shows no effect |

## 5. Intent vs Execution Split

Use the two-stage result explicitly:

- `unmatched_intent`: agent probably did not attempt the checkpoint.
- intent matched but verifier failed: agent attempted it but did not achieve the state.
- verifier uncertain or missing screenshot: evidence gap, not a direct planning failure.

This split is essential for separating planning failure from execution/environment failure.

## 6. Suggested Output

```json
{
  "label": "normal|abnormal|uncertain",
  "type": "planning_failure",
  "subtype": "missing_required_checkpoint",
  "confidence": 0.0,
  "missing_checkpoints": [],
  "first_error_step": -1,
  "evidence": [],
  "related_anomalies": []
}
```

## 7. Implementation Plan

1. Build `src.evaluator.planning_failure_baseline` or add a focused module under `src.common`.
2. Consume `intent_matches`, `verification_report`, and `state_sequence` only.
3. Treat missing evidence as `uncertain`, not failure.
4. Add unit tests for unmatched intent, verifier failure, premature finish, and fail-to-terminate.
5. Add labeled regression cases before tuning thresholds.
