# 规划失效异常判定技术方案

版本：v1.1
更新日期：2026-07-17
状态：第一版已实现，并已接入 `src.evaluator.repeated_baseline` 单跑和批跑输出

## 1. 目标

检测任务层面的规划失败，包括必要 checkpoint 缺失、目标错误、顺序错误、提前终止，以及目标完成后未停止。

规划失效不同于单步动作失败。它关注实际轨迹是否持续推进用户任务和必要 checkpoint。

## 2. 当前已具备证据

新基线已经产出第一版规划失效模块需要的证据：

| 输出 | 用途 |
|---|---|
| `_checkpoints` | 预期子目标 |
| `intent_matches.json` | 实际 Agent 意图是否召回到 checkpoint 候选 |
| `checkpoint_alignments.json` | 每个 checkpoint 的执行候选步骤 |
| `verification_report.json` | checkpoint 是否被截图/VLM 证明达成 |
| `state_sequence.json` | 状态进展和视觉/OCR 边界 |
| `repeated_prediction.json` | 重复操作或循环症状 |

## 3. 当前分类器

规划失效模块位于 `src.evaluator.planning_failure`，消费当前基线输出，不依赖旧 `src.oracle`。`start.sh` 调用 `src.evaluator.repeated_baseline` 后会自动生成 `planning_failure_result.json`，批跑汇总会增加规划失效标签、子类型和置信度。

```text
checkpoints
  + intent_matches
  + checkpoint verification
  + state_sequence
  + repeated_prediction
  -> planning_failure_result
```

## 4. 子类型

| 子类型 | 证据 |
|---|---|
| `missing_required_checkpoint` | required checkpoint 未召回或未达成 |
| `premature_termination` | required checkpoint 达成前出现 finished |
| `fail_to_terminate` | checkpoint 已达成但 Agent 继续无进展操作 |
| `objective_or_plan_mismatch` | intent match 长期指向错误目标或 forbidden state |
| `execution_blocked` | Agent 有正确意图，但 VLM/AB/state 证据显示未生效 |

## 5. 意图与执行拆分

必须显式利用两阶段结果：

- `unmatched_intent`：Agent 大概率没有尝试该 checkpoint。
- intent matched 但 verifier failed：Agent 尝试了，但没有达成状态。
- verifier uncertain 或缺截图：证据不足，不能直接归为规划失败。

这一区分是拆分规划失败、执行失败和环境失败的关键。

## 6. 当前输出

```json
{
  "label": "normal|abnormal|uncertain",
  "type": "planning_failure",
  "subtype": "missing_required_checkpoint",
  "confidence": 0.0,
  "missing_checkpoints": [],
  "first_error_step": -1,
  "evidence": [],
  "related_anomalies": []
}
```

## 7. 已完成

1. 新增 `src.evaluator.planning_failure`。
2. 接入 `run_repeated_baseline`，输出 `planning_failure_result.json`，并写入 `baseline_result.json`。
3. 接入 `run_repeated_baseline_batch`，在 `batch_result.json.results[*]` 中增加规划失效摘要字段。
4. 增加 unmatched intent、verifier failure、premature finish、fail-to-terminate 单测。

## 8. 后续校准

1. 在人工标注集上统计各 subtype 的 precision/recall。
2. 校准 `completion_threshold` 和 `fail_to_terminate_extra_steps`。
3. 根据真实样本补充 `objective_or_plan_mismatch` 的高置信证据，目前第一版不主动输出该 subtype。
