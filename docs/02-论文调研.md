# 论文调研：GUI Agent 执行评估

> 调研主题：多模态 GUI Agent 执行能力的自动化判定，侧重操作前后截图对比的异常检测。

---

## 核心论文列表

### 1. VeriGUI: Don't Act Blindly — Action-Effect Verification and Self-Correction

| 属性 | 值 |
|------|-----|
| 出处 | ACL 2026 |
| arXiv | [2604.05477](https://arxiv.org/abs/2604.05477) |
| 开源 | ❌ 未公开代码 |
| 模型 | Qwen2.5-VL-3B/7B |
| 训练 | Stage 1: Robust SFT(GPT-4o 标注) → Stage 2: GRPO 强化学习 |

**核心思路**：TVAE 闭环框架（Think → Verify → Action → Expected Effect）

- 每步操作前预测"预期屏幕效果"
- 操作后截图对比预期效果
- 二元判定：SUCCESS（继续）/ NO_CHANGE（诊断+恢复）

**五类合成失败训练**：
- 坐标偏移（30%）、动作类型错误（25%）、目标误识别（20%）、时序错误（15%）、空点击（10%）

**关键发现**：72.3% 的 Agent 失败来自重复无效操作导致超时

---

### 2. TrajAD: Trajectory Anomaly Detection for Trustworthy LLM Agents

| 属性 | 值 |
|------|-----|
| 出处 | arXiv 2026 |
| arXiv | [2602.06443](https://arxiv.org/abs/2602.06443) |
| 开源 | ❌ 未公开代码 |
| 模型 | Qwen3-4B (纯文本 LLM) |
| 训练 | QLoRA LoRA r=8, α=16，影响 1.8% 参数 |

**三大异常类型**：
- **任务失败**：推理错误 / 执行错误
- **过程低效**：循环回路 / 绕行
- **不当延续**：冗余操作 / 未感知终止

**关键结论**：4B 微调 (F1 81.8%) > 8B 零样本 (F1 67.9%) — 专用过程审计模型 > 通用大模型

**与项目方案关联**：轨迹差分三分类参考了 TrajAD 的异常分类体系

---

### 3. GUI-Critic-R1: Look Before You Leap — Pre-Operative Error Diagnosis

| 属性 | 值 |
|------|-----|
| 出处 | arXiv 2025 |
| 类型 | 执行前检查 |

**核心思路**：在 Agent 执行每个操作**之前**预判是否会产生错误

- 对操作的目标元素、动作参数进行正确性诊断
- 属于 Pre-Action 范式，与 VeriGUI 的 Post-Action 互补

---

### 4. GUI-SHEPHERD: Reliable Process Reward and Verification for Long-Sequence GUI Tasks

| 属性 | 值 |
|------|-----|
| 出处 | arXiv 2025 |
| 类型 | 步骤级过程验证 |

**核心思路**：
- 52k 人标注交互数据 + 逐步骤奖励模型
- 为长序列任务提供步骤级 process reward
- 启发：检查点达成率评分体系

---

### 5. V-Droid: Verifier-Driven Mobile GUI Agent

| 属性 | 值 |
|------|-----|
| 出处 | MobiCom 2026 |
| 类型 | 验证器驱动代理 |

**核心思路**：将 MLLM 从"动作生成器"重构为"动作验证器"

- 候选动作评估 + 仅预填充加速
- 验证器视角而非执行器视角

---

### 6. VeriSafe Agent: Logic-based Action Verification for Mobile GUI

| 属性 | 值 |
|------|-----|
| 出处 | arXiv 2025 [2503.18492](https://arxiv.org/abs/2503.18492) |
| 类型 | 形式化逻辑验证 |

**核心思路**：将自然语言指令转化为可验证规范，在动作执行前做确定性检查

- 不适合复杂动态场景，但可作为简单规则层的补充
