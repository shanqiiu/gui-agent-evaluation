# GUI Agent 重复动作异常判定技术方案

> 版本：v0.1 · 2026-07-06  
> 目标：针对 GUI Agent 执行轨迹中的重复点击、重复输入、重复滑动、重复跳转等异常，建立可落地的自动判定方案。  
> 输入边界：使用实际 Agent 轨迹中的截图序列、动作类型、实际执行坐标、输入文本、动作描述和达尔文单步/E2E 判定结果；不依赖人工预置的期望点击坐标。

---

## 1. 问题定义

重复动作是指 Agent 在任务执行过程中，对同一页面状态、同一目标控件或同一语义目标反复执行相同或等效动作，但没有产生新的任务进展。

需要区分两类情况：

| 类型 | 说明 | 判定态度 |
|------|------|----------|
| 合理重复 | 翻页、滚动列表、重试加载、连续删除字符、多选多个同类元素 | 不判异常或低风险提示 |
| 异常重复 | 对同一控件反复点击、同一文本反复输入、A/B 页面来回跳转、重复执行已完成步骤 | 判为重复动作异常 |

核心判定原则：

```
动作等效 + 目标等效 + 页面状态无有效进展 + 不属于合理重复白名单
    => 重复动作异常
```

## 2. 可用输入

### 2.1 轨迹数据

来自 Agent 执行日志或 FuncOracleCheck 的 `seq_info` / `actionList`：

| 字段 | 用途 |
|------|------|
| `image_relative_path` / `img` | 当前步骤截图 |
| `planning_output.parsed_action.action_type` / `operType` | 动作类型，click、type、scroll、wait、finished 等 |
| `start_box` / `startBox` | 实际执行坐标，不是预期坐标 |
| `end_box` / `endBox` | 滑动/拖拽终点坐标 |
| `text` | 输入文本或 Agent 自述动作意图 |
| `direction` | 滑动方向 |

### 2.2 达尔文判定产物

| 产物 | 用途 |
|------|------|
| `ab_pages_result` | 单步前后页面跳转是否符合预期 |
| `ActionDescription` | VLM 对被操作控件的语义描述 |
| `page_description` | 页面语义摘要 |
| `intention` / `*_intention_step` | E2E 意图和子步骤达成情况 |

### 2.3 派生特征

| 特征 | 计算方式 | 作用 |
|------|----------|------|
| 动作规范化签名 | `action_type + text + direction` 归一化 | 识别同类动作 |
| 坐标邻近度 | 坐标按屏幕宽高归一化后计算距离 | 识别同一目标区域 |
| 控件语义相似度 | `ActionDescription` embedding / LLM 分类 | 识别同一控件或同类控件 |
| 页面状态相似度 | pHash/SSIM/OCR/page_description embedding | 判断页面是否停滞 |
| 检查点进展 | 达尔文 E2E 子步骤是否新增达成 | 判断是否推进任务 |

## 3. 判定架构

```
执行轨迹
  │
  ├─ Step Normalizer
  │    └─ 统一动作类型、坐标、文本、页面 ID
  │
  ├─ Target Resolver
  │    └─ 基于坐标局部截图 + VLM 描述目标控件
  │
  ├─ State Comparator
  │    └─ 判断相邻/跨步页面是否语义等价
  │
  ├─ Progress Tracker
  │    └─ 判断任务检查点是否有新增达成
  │
  └─ Repetition Detector
       └─ 输出重复动作区间、严重度、证据
```

---

## 4. 核心算法

### 4.1 动作等效判定

两个动作 `a_i`、`a_j` 满足以下条件时认为动作等效：

| 动作类型 | 等效条件 |
|----------|----------|
| click / long_press | 动作类型一致或同属点击类；坐标距离小于阈值；目标控件语义相似 |
| type / set_text | 输入目标控件相同；输入文本相同或高度相似 |
| scroll / swipe / drag | 方向一致；起止区域相近；页面列表内容无明显新增 |
| wait / do-nothing | 连续等待且页面状态无变化 |

建议阈值：

| 阈值 | 默认值 | 说明 |
|------|--------|------|
| 坐标归一化距离 | `<= 0.06` | 以屏幕对角线归一化 |
| 控件语义相似度 | `>= 0.85` | embedding cosine 或 LLM 二分类 |
| 文本相似度 | `>= 0.95` | 输入文本需更严格 |
| 页面语义相似度 | `>= 0.88` | page_description embedding |
| pHash 汉明距离 | `<= 8` | 页面近似不变 |

### 4.2 页面无进展判定

页面无进展不是简单的“截图完全相同”，需要同时看视觉、语义和任务进度：

```
no_progress(i, j) =
    page_similarity(i, j) >= T_page
    AND ocr_delta(i, j) is small
    AND no new checkpoint satisfied
    AND Darwin single-step result not clearly positive
```

其中达尔文结果的使用方式：

| 达尔文结果 | 重复判定影响 |
|------------|--------------|
| `符合预期` | 降低重复异常置信度，除非后续又回到同一状态 |
| `不符合预期` | 提高重复异常置信度 |
| `无法判定` | 不直接作为异常证据，仅作为弱信号 |

### 4.3 连续重复检测

用于识别“连续点同一个按钮”“连续输入同一文本”等明显重复。

伪代码：

```python
for each step i:
    window = steps[i-k:i]
    repeated = count(equivalent_action(step, steps[i]) for step in window)
    stagnant = no_progress(last_distinct_state, current_state)
    if repeated >= n and stagnant and not in_whitelist(step):
        emit_repeated_action(start, end, confidence)
```

默认规则：

| 场景 | 触发条件 |
|------|----------|
| 点击重复 | 3 步内对同一控件点击 >= 2 次，且页面无进展 |
| 输入重复 | 同一输入框重复写入相同文本 >= 2 次 |
| 等待重复 | 连续 wait >= 3 次，页面无加载状态变化 |
| 滑动重复 | 同方向滑动 >= 4 次，列表无新增内容或回到旧内容 |

### 4.4 非连续循环重复检测

用于识别 A→B→A、A→B→C→A 这类状态循环。

```
state_signature = hash(
    normalized_page_description,
    selected_tab,
    top_ocr_tokens,
    coarse_layout_hash
)

action_signature = hash(
    action_type,
    target_widget_description,
    normalized_text_or_direction
)
```

当出现以下模式时触发：

```
state_signature[i] == state_signature[j]
AND action_signature[i] == action_signature[j]
AND no checkpoint progress between i and j
AND j - i <= loop_window
```

默认 `loop_window=8`。

## 5. 合理重复白名单

避免误杀的关键是先识别合理重复场景。

| 场景 | 豁免条件 |
|------|----------|
| 列表滚动搜索 | 每次滑动后 OCR 新增内容比例 > 20% |
| 加载失败重试 | 页面出现“加载失败/重试/网络异常”，点击重试 <= 2 次 |
| 多选同类项目 | 坐标不同，目标控件语义同类但实例不同 |
| 删除/编辑文本 | 输入框文本长度持续变化 |
| 轮播图/分页 | 页码、焦点图、tab 选中态发生变化 |
| 权限/弹窗处理 | 重复点击是为关闭不同弹窗实例 |

## 6. 输出格式

建议新增独立判定结果 `repeated_action_result`：

```json
{
  "label": "abnormal",
  "type": "repeated_action",
  "severity": "high",
  "confidence": 0.91,
  "ranges": [
    {
      "start_step": 4,
      "end_step": 6,
      "action_type": "click",
      "target": "点击播放页右侧点赞按钮",
      "evidence": [
        "步骤4和步骤6点击坐标归一化距离为0.02",
        "目标控件语义相似度0.93",
        "页面语义相似度0.91",
        "期间无新增检查点达成"
      ]
    }
  ],
  "summary": "Agent 在播放页对同一点赞按钮重复点击，页面状态和任务进度未发生有效变化。"
}
```

严重度定义：

| 严重度 | 条件 |
|--------|------|
| low | 重复 2 次，但最终任务完成 |
| medium | 重复 2-3 次，造成步骤浪费或短暂回退 |
| high | 重复 >= 3 次，导致任务失败、循环或无法终止 |

## 7. 与现有项目集成

### 7.1 离线评估路径

在达尔文 E2E 判定完成后，读取：

- 原始 `seq_info`
- `ab_pages_result`
- `page_des`
- `action_des`
- `intention` / `*_intention_step`

然后执行重复动作检测，作为综合评估中的“执行效率”和“异常行为”子项。

### 7.2 在线告警路径

在线执行时只做轻量规则：

| 实时信号 | 动作 |
|----------|------|
| 同坐标同动作连续 2 次且截图近似不变 | 告警：疑似重复点击 |
| 同页面状态重复出现 3 次 | 告警：疑似循环 |
| 连续 wait 超阈值 | 告警：疑似卡死 |

完整语义判定仍放在线下，避免在线推理成本过高。

## 8. 技术穿刺计划

### 阶段一：规则基线

目标：不用训练模型，先跑通 80% 明显重复 case。

交付：

- 轨迹解析器：统一 `seq_info` / `actionList`
- 坐标归一化与动作签名
- pHash + OCR token 页面相似度
- 连续重复和短循环检测

### 阶段二：达尔文结果增强

目标：降低误报，识别“动作重复但状态有进展”的合理场景。

交付：

- 接入 `ab_pages_result`
- 接入 `ActionDescription`
- 接入 `page_description`
- 输出结构化证据链

### 阶段三：小样本标定

目标：用人工标注集标定阈值。

建议数据：

| 类型 | 数量 |
|------|------|
| 明显重复点击 | 30 |
| 重复输入 | 20 |
| 循环跳转 | 30 |
| 合理滚动/翻页 | 30 |
| 合理重试/弹窗处理 | 20 |

评估指标：

| 指标 | 目标 |
|------|------|
| Precision | >= 0.85 |
| Recall | >= 0.75 |
| 误杀合理滚动率 | <= 0.10 |

## 9. 风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| 动态页面导致截图差异大 | 漏判重复 | 加入 OCR 和控件语义相似度 |
| 同类控件多实例相似 | 误判多选为重复 | 引入坐标区域和列表 item 实例差异 |
| 页面局部动画影响 pHash | 页面相似度不稳定 | 使用局部 mask 或 OCR/page_description 兜底 |
| 达尔文返回“无法判定” | 证据不足 | 不把“无法判定”作为强异常信号 |
| 合理重试被误杀 | 影响评估公正性 | 建立加载失败/网络异常白名单 |

---

## 10. 结论

重复动作判定不应只看 action history 中是否出现相同动作，而应联合判断“动作是否等效、目标是否等效、页面是否无进展、任务检查点是否无新增达成”。该方案可以优先以规则和达尔文判定结果落地，后续再用标注集对阈值和语义分类器进行校准。
