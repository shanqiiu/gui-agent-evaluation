# 重复操作异常判定技术方案

版本：v1.0
更新日期：2026-07-16
状态：与当前新基线对齐的专题方案

## 1. 目标

在真实 Agent 执行轨迹中检测重复 GUI 操作，且不引入 App 或业务场景硬编码。

重复操作指：Agent 在相同或等效状态/目标上重复执行相同或等效动作，但页面、OCR、视觉状态或 checkpoint 进度没有有效推进。

## 2. 当前实现

当前基线由以下模块实现：

- `src.common.repeated_action_detector`
- `src.evaluator.repeated_baseline`
- `src.evaluator.state_evidence`

输入：

- hydrated `payload`
- `ab_report`
- `verification_report`
- `state_sequence`

输出：

- `repeated_prediction.json`
- `baseline_result.json.repeated_prediction`

## 3. 使用证据

| 证据 | 用途 |
|---|---|
| action type/text/coordinates | 判断动作等效 |
| AB label 和页面描述 | 判断单步是否产生预期进展 |
| state transitions | 判断页面/状态是否变化 |
| checkpoint verification | 判断任务进度是否新增 |
| visual evidence | pHash、SSIM、pixel diff、变化区域 |
| OCR/rawPage | 文本和结构状态变化 |

非 GUI 动作会被忽略，包括用户回复、播报、语音播报、TTS、assistant/system/reply/speak 等。

## 4. 判定原则

```text
动作等效
+ 目标等效或状态相同
+ 无状态转移
+ 无 checkpoint 进展
+ 不属于允许的合理重复模式
=> repeated_action
```

检测器不能仅凭动作文本相似就输出高置信重复异常。

## 5. 合理重复

以下场景不应直接判异常，除非有明确无进展证据：

- 滚动列表且出现新内容
- 加载或网络错误后的有限重试
- 关闭不同弹窗实例
- 对不同坐标/实例进行多选
- 文本编辑过程中输入框内容持续变化
- 页面加载中的 do-nothing/wait

## 6. 当前输出格式

```json
{
  "label": "normal|abnormal",
  "type": "repeated_action",
  "severity": "none|low|medium|high",
  "confidence": 0.0,
  "ranges": [],
  "summary": "...",
  "metrics": {
    "action_count": 0,
    "repeated_range_count": 0
  }
}
```

## 7. 已知限制

- 目标等效仍部分依赖动作文本和坐标。
- 检测效果高度依赖 `state_sequence` 和 checkpoint verification 的质量。
- OCR 与视觉阈值仍需在标注集上校准。
