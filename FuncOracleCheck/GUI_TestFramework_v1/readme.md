# HarmonyAPPTest Framework 使用文档

## 1. 测试模式
测试模式为框架默认运行模式，核心配置需在 `config.py` 文件中完成，具体配置与使用流程如下：

### 1.1 核心配置项（config.py）
| 配置项          | 配置要求与说明                                                                 |
|-----------------|------------------------------------------------------------------------------|
| `PREDICATE_MODE` | 默认为测试模式，无需手动修改（保持默认值即可启用测试模式）                       |
| 滑窗模式        | 必须指定为 `LLM` 或 `VLM` 二选一（根据测试场景选择对应的模型类型）               |
| 模型信息        | 需补充 `LLM` 或 `VLM` 对应的模型名称与访问URL（确保模型可正常调用）       |
| `DATA_DIR`      | 配置测试样本文件夹路径（框架将从该路径读取待测试样本文件）                       |
| `SAVE_RESULT`   | 布尔值配置（`True`/`False`），用于控制是否保存测试后的判定结果文件               |

### 1.2 测试模式分类与执行
#### 1.2.1 单实例测试
- 适用场景：单次测试单个样本，快速验证模型逻辑或测试流程正确性
- 执行文件：`framework.py`
- 操作步骤：
  1. 完成 `config.py` 中上述核心配置；
  2. 直接运行 `framework.py`，框架将自动加载配置并执行单实例测试。

#### 1.2.2 Benchmark 测评
- 适用场景：批量测试多组样本，统计模型准确率、耗时等综合测评指标
- 执行文件：`framework_batch_eval.py`
- 结果统计：批量测评完成后，运行 `eval.py` 文件可自动统计测评结果。
